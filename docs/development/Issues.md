# KalaKriti — Issues Log

> Issues found during local testing. Each entry has three fields: what was seen, why it happened (simple language), and how it was fixed.

---

## Template

```
### ISSUE-XXX: Short title

**Phase:** Phase N  
**Date:** YYYY-MM-DD  
**Status:** Open / Fixed

**Observation**  
What the user saw — described in plain terms.

**Root Cause**  
Why it happened — one or two sentences, simple language, no jargon.

**Solution**  
The fix applied or the best approach chosen.

---
```
